# Coding principles

## General
- DRY — avoid duplication of logic
- KISS — prefer simple solutions
- YAGNI — do not build unused abstractions
- SLAP — single level of abstraction per function

## Clean code
- Meaningful names
- Small functions
- Clear control flow
- No magic numbers or strings
- Consistent formatting
- Explicit error handling

## Readability
- Stepdown rule: code is read top-down
- Fail fast when assumptions are violated
